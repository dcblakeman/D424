Why this deployment works

  • Publishes over HTTPS from a managed cloud endpoint.
  • Uses GitLab CI/CD to redeploy automatically after a push to master.
  • Distributes the signed Android build from a public release link.
  • Keeps the code repository and deployed site connected in one workflow.

Core Features

  • Term, course, assessment, note, and instructor management
  • SQLite-backed local persistence and validation
  • Reminder scheduling and reporting workflows
  • Android install package linked directly from the deployment page

Deployment Automation

The .gitlab-ci.yml pipeline runs the PowerShell build script in a container image to generate the published public site. A companion Dockerfile uses the same generated site and serves it through Nginx for portable container-based hosting.
